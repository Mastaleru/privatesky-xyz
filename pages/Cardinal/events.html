This page is intending to display how Events are written in this framework and the ways of how to use them inside components and controllers. In Cardinal framework usage of events is encouraged in order to pass data or trigger actions from view to controllers and vice-versa. All implementations should respect the bubbling and capturing principles (external link) . Cardinal events are wrapped native custom events. The wrapping was necessary for a better handling of events with the same name and to avoid unwanted conflicts. Each event has a method called getEventType() which is returning the event type. The event type is different from event name and it should be carefully chosen in the development process. In pure native custom events, data is passed through the detail property. As a convention, cardinal events are storing the data on data property. The following code snippet is an example of cardinal event. import {EVENTS_TYPES} from "../utils/constants"; const EVENT_TYPE = EVENTS_TYPES.PSK_BUTTON_EVT; export default class PskButtonEvent extends CustomEvent { public data: any; public getEventType = function() { return EVENT_TYPE; }; constructor(eventName: string, eventData: any, eventOptions: EventInit) { super(eventName, eventOptions); this.data = eventData; } } For using an event, it should be instantiated with the following parameters:   eventName - required   eventData - optional   eventOptions - optional

The default event options are:  bubbles:false  composed:false  cancelable:false

The event is dispatched using elem.dispatchEvent(event) to the specified target. In the following code snippet PskButtonEvent is used in psk-button to trigger an event. let pskButtonEvent = new PskButtonEvent(this.eventName, this.eventData, { bubbles: true, composed: true, cancelable: true }); let eventDispatcherElement = this.htmlElement; if (this.eventDispatcher) { if (ACCEPTED_DEFAULT_DISPATCHERS.indexOf(window[this.eventDispatcher]) !== -1) { eventDispatcherElement = window[this.eventDispatcher]; } } eventDispatcherElement.dispatchEvent(pskButtonEvent); In controllers, usually, the event target is the HTMLElement parameter of the constructor. In most of the cases it will be used to listen for bubbling events.
