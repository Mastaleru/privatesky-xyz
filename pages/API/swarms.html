APIs for creating swarm and flow type swarms Creates a swarm description named swarmTypeName and register in the specific name space. $$.swarm.describe(swarmTypeName, description); $$.flow.describe(swarmTypeName, description); swarmTypeName A string representing the type of the swarm defined.

description An object that represents the swarm's definition. In the example below, a swarm with the name echo is described. $$.swarms.describe("echo", { say: function (message) { this.return(null, 'Echo ' + message); } }); Calls the specified function from the specified swarm. $$.swarm.start(swarmTypeName, ctor, ...params) $$.flow.start(swarmTypeName, ctor, ...params) swarmTypeName A string representing the type of the swarm defined.

ctor The function called.

...params The arguments the function ctor takes. Calls the specified function from the specified swarm with the specified context. $$.swarm.startWithContext(context, swarmTypeName, ctor, ...params); $$.flow.startWithContext(context, swarmTypeName, ctor, ...params); context The string passed to the homeSecurityContext metadata in the swarm instance.

swarmTypeName A string representing the type of the swarm defined.

ctor The function called.

...params The arguments the function ctor takes. Continue a swarm with the provided serialization. $$.swarm.continue(swarmTypeName, initialValues); $$.flow.continue(swarmTypeName, initialValues); swarmTypeName A string representing the type of the swarm defined.

initialValues The swarm serialization. Starts a swarm based on a description without registering and giving a name. $$.swarm.inline(description, ctor, ...args); $$.flow.inline(description, ctor, ...args); description The swarm description.

swarmTypeName A string representing the type of the swarm defined.

ctor The function called.

...args The arguments the function ctor takes. Note that the most used APIs are describe and start.
