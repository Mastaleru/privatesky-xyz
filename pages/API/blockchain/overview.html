The blockchain module allows the creation of custom blockchains, by allowing the programmer to choose the history storage, the consensus algorithm as well as the world state cache type that the created blockchain uses. The blockchain module provides two methods for creating blockchains. The first, createBlockchain(), creates a singleton blockchain instance assigned to the global variable $$.blockchain, while the method createABlockchain returns an instance of Blockchain class (details here). The two methods receive the same parameters and, as a result, only createABlockchain is described. The createBlockchain() returns a createABlockchain(worldStateCache, historyStorage, consensusAlgorithm, signatureProvider, loadDefaultConstitution, forcedBoot); worldStateCache The world state cache strategy used.

historyStorage The history storage strategy used.

consensusAlgorithm The consensus algorithm used.

signatureProvider The signature provider.

loadDefaultConstitution A boolean variable used for deciding whether the default constitution is used. Default true

forcedBoot A boolean variable used to decide whether another instance of blockchain is overwritten. Default true

 A Blockchain instance. In the following example, the strategies used for creating a blockchain are created before retrieving the Blockchain instance. const bm = require("blockchain"); let worldStateCache = bm.createWorldStateCache("memory"); let historyStorage = bm.createHistoryStorage("memory"); let consensusAlgorithm = bm.createConsensusAlgorithm("direct"); let signatureProvider = bm.createSignatureProvider("permissive"); bm.createABlockchain(worldStateCache, historyStorage, consensusAlgorithm, signatureProvider);
