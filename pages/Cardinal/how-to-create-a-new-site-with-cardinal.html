On this page can be found all the information about how to creat a new skeleton site using Cardinal.Js module, cardinal-updater.

Git - using git, you can clone the project from Github, but is also necessary in the creation process.

Node.Js with NPM - npm is required to be able to run the installer, to install and run the created project.

cardinal-updater - this module is the one who is responsable with the creation of the application skeleton.

Download the archive with the project or clone it to your machine, then install the module globally by running:

$ ~/path/to/cardinal-updater> npm install -g

Display the help section: command cardinal -h or cardinal --help or cardinal help.

$ cardinal -h Usage: cardinal [options] [command] Options: --version output the version number -m, --menu [type] [type] set where to dock the application menu. Possible values (left|top). The default value is left. (default: left) -h, --help output usage information Commands: init [dir] creates an instance of Cardinal.Js withing a directory named [dir]. Default: Cardinal-SSApp update updates an existing instance of Cardinal.Js. This must be executed on the root of the project. E.g: $ path/to/Cardinal-SSApp> cardinal update help [cmd] display help for [cmd]

Example of creation of a skeleton: dir = "myCardinalApp", --menu=top. Run the following command:

$ cardinal init myCardinalApp --menu=top

By running this command a directory named "myCardinalApp" was created.

Inside it there were added the builds from PrivateSky-Release, which includes the source files for BlockChain, CSB and Two-way binding and also the build which includes all the web-components developed within pskwebcomponents module.

The build is made within Cardinal module and the built source file are brought to the skeleton.

Node: You can run also cardinal init with no problems, the differences are that the default values for menu and dir will be taken as parameters for the skeleton creation process.

After all the skeleton is created, use the following commands:

$ cd myCardinalApp $ npm install $ npm run web (make sure you dont have port: 8000 already used, but you can change this in package.json file.)

To update an instance of Cardinal.Js framework, you have to be inside the root of the web-site, (where you can run npm run web) and to run the command:.

$ cardinal update

The cardinal-updater module will bring the latest updates from PrivateSky-Release and Cardinal.Js to your own project.
